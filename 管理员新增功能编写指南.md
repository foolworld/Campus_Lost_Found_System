# 管理员新增功能编写指南

> **说明**：本指南核心交付为**后端代码提示词**，前端提示词为补充内容，用于完整实现功能时参考。所有新增代码需与现有项目结构兼容，尽量不破坏原有功能，确保分支合并顺利。

## 项目概述

本项目是基于 **Flask 3.x** + **SQLite** + **Bootstrap 5** 构建的校园失物招领系统，采用单体应用架构，所有后端逻辑集中在 `app.py` 文件中。

class Post(db.Model):

    post_type = db.Column(db.String(10), default='found')
    status = db.Column(db.String(20), default='pending')
    created_at = db.Column(db.DateTime, default=datetime.datetime.now)
```

## 功能一：批量删除用户

### 需求分析
- 在用户管理页面勾选多个用户
- 点击批量删除按钮一次性删除
- 不能删除管理员账号
- 删除用户时需一并删除其发布的帖子

### 实现方案

#### 1. 数据库模型
无需新增模型，使用现有 `User` 和 `Post` 模型。

#### 2. 后端路由设计

**批量删除用户路由**：

| 属性 | 值 |
|------|------|
| 路由 | `/admin/users/batch-delete` |
| 请求方法 | POST |
| 权限 | 仅管理员 (`current_user.role == 'admin'`) |
| 请求参数 | `user_ids` (列表，选中的用户ID) |

#### 3. 关键代码提示词

**生成后端代码提示词**：

```

2. 权限验证：使用 @login_required 装饰器，且仅管理员可访问（验证 current_user.role == 'admin'）
3. 请求参数：从 request.form 获取 user_ids（多个ID用逗号分隔，例如 "1,2,3"）

   - 使用 Post.query.filter_by(author_id=user_id).delete() 删除该用户的所有帖子
   - 使用 db.session.delete(user) 删除用户
   - 最后 db.session.commit()
5. 返回结果：
   - 删除成功后 flash 提示成功信息，包含成功删除的数量
   - 如果有管理员被跳过或用户不存在，flash 提示警告信息
   - 重定向到管理后台用户管理页（url_for('admin_dashboard', tab='users')）
6. 代码风格：与现有代码保持一致，使用 Flask-SQLAlchemy 的标准操作方式

## 功能三：违规举报处理

### 需求分析
- 用户可以在信息详情页举报违规信息
- 管理员在管理后台查看举报列表
- 管理员可以处理举报（确认违规/驳回举报）
- 确认违规后可以删除该信息并通知用户

### 实现方案

#### 1. 数据库模型设计

需要新增 `Report` 模型：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Integer | 主键，自增 |
| post_id | Integer | 被举报的帖子ID（外键） |
| reporter_id | Integer | 举报人ID（外键） |
| reason | Text | 举报原因 |
| status | String(20) | 举报状态（pending/handled） |
| result | String(20) | 处理结果（violation/rejected） |
| handled_by | Integer | 处理人ID（外键，可为空） |
| created_at | DateTime | 举报时间 |
| handled_at | DateTime | 处理时间 |

#### 2. 后端路由设计

**举报相关路由**：

| 功能 | 路由 | 请求方法 | 权限 | 说明 |
|------|------|----------|------|------|
| 提交举报 | `/post/<post_id>/report` | POST | 登录用户 | 用户举报违规信息 |
| 举报列表 | `/admin/reports` | GET | 管理员 | 查看所有举报 |
| 处理举报 | `/admin/report/<report_id>/handle` | POST | 管理员 | 处理举报（确认/驳回） |

## 代码兼容性规范

### 命名规范
- 路由名称使用 snake_case 风格（如 `batch_delete_users`）
- 函数名称使用 snake_case 风格
- 变量名称使用 snake_case 风格
- 模型类使用 PascalCase 风格

### 权限验证规范
- 所有管理功能必须使用 `@login_required` 装饰器
- 管理员操作必须验证 `current_user.role == 'admin'`
- 操作成功/失败使用 `flash()` 提示用户

### 数据库操作规范
- 使用 `db.session.add()` 添加新记录
- 使用 `db.session.delete()` 删除记录
- 使用 `db.session.commit()` 提交事务
- 查询优先使用 `get_or_404()` 处理不存在的记录

### 错误处理规范
- 使用 `flash()` 提示用户操作结果
- 使用 `redirect()` 重定向到合适的页面
- 对用户输入进行验证，防止无效数据

### 安全规范
- **CSRF 防护**：当前项目未集成 Flask-WTF，新增的 POST 表单存在 CSRF 风险。生产环境建议集成 Flask-WTF 并添加 CSRF token。开发阶段可暂时忽略，但部署前必须补充 CSRF 防护。
- **批量操作安全**：批量操作前必须验证参数不为空，避免空操作或错误操作。
- **权限验证**：所有管理功能必须验证管理员权限，防止越权访问。
