# 拆分后的目录结构

## 项目概述

本项目是基于 Flask 框架开发的校园失物招领系统。为了遵循单一职责原则，将原有的单文件 `app.py` 拆分为多个模块，形成清晰的目录结构。

## 目录结构

```
Campus_Lost_Found_System/
├── app.py              # 应用主入口
├── config.py           # 应用配置与初始化
├── models.py           # 数据库模型定义
├── services.py         # 业务逻辑层
├── utils.py            # 工具函数
├── routes/             # 路由模块
│   ├── __init__.py     # 路由注册
│   ├── auth.py         # 用户认证路由
│   ├── main.py         # 首页与搜索路由
│   ├── post.py         # 失物信息路由
│   ├── comment.py      # 评论与回复路由
│   ├── admin.py        # 管理员后台路由
│   ├── notice.py       # 公告路由
│   ├── notification.py # 通知路由
│   └── report.py       # 举报路由
├── templates/          # Jinja2 模板
├── static/             # 静态资源
│   ├── css/            # 样式文件
│   └── js/             # JavaScript 文件
└── 文档/               # 项目文档
```

## 文件说明

### 根目录文件

#### app.py
- **作用**：应用主入口文件，负责创建 Flask 应用实例、初始化数据库、注册路由
- **功能**：
  - 调用 `create_app()` 创建应用
  - 调用 `init_login_manager()` 初始化登录管理器
  - 调用 `register_routes()` 注册所有蓝图
  - 启动时创建数据库表和示例数据

#### config.py
- **作用**：应用配置与初始化模块
- **功能**：
  - 定义 `create_app()` 函数，配置 Flask 应用参数（密钥、数据库 URI、上传目录等）
  - 定义 `init_login_manager()` 函数，初始化 Flask-Login 扩展
  - 初始化 SQLAlchemy 数据库连接

#### models.py
- **作用**：数据库 ORM 模型定义
- **功能**：
  - 定义 `User` 模型：用户信息（用户名、密码哈希、角色）
  - 定义 `Post` 模型：失物招领信息（标题、描述、地点、时间、图片等）
  - 定义 `Comment` 模型：评论与回复（支持层级回复结构）
  - 定义 `Report` 模型：举报信息（举报原因、处理状态）
  - 定义 `Notification` 模型：通知信息（类型、内容、已读状态）
  - 定义 `Notice` 模型：系统公告（标题、内容、发布状态）

#### services.py
- **作用**：业务逻辑层，封装核心业务功能
- **功能**：
  - `search_posts()`：搜索失物信息（支持标题、描述、地点搜索）
  - `find_similar_posts()`：查找相似帖子（基于 TF-IDF 和余弦相似度）
  - `create_notification()`：创建用户通知
  - `create_sample_data()`：生成示例数据（首次启动时自动调用）

#### utils.py
- **作用**：工具函数模块
- **功能**：
  - `cut_text()`：使用 jieba 对文本进行分词
  - `tfidf_vectorize()`：计算文本的 TF-IDF 向量
  - `cosine_similarity()`：计算两个向量的余弦相似度

### routes/ 目录

#### routes/__init__.py
- **作用**：路由注册模块
- **功能**：
  - 导入所有路由蓝图
  - 定义 `register_routes()` 函数，将所有蓝图注册到 Flask 应用

#### routes/auth.py
- **作用**：用户认证路由
- **功能**：
  - `/register`：用户注册
  - `/login`：用户登录
  - `/logout`：用户退出登录

#### routes/main.py
- **作用**：首页与搜索路由
- **功能**：
  - `/`：首页，展示最新失物信息
  - `/search`：搜索页面，支持关键词搜索

#### routes/post.py
- **作用**：失物信息路由
- **功能**：
  - `/post/<int:post_id>`：帖子详情页
  - `/post/publish`：发布新信息
  - `/post/<int:post_id>/edit`：编辑帖子
  - `/post/<int:post_id>/delete`：删除帖子
  - `/my-posts`：我的发布列表

#### routes/comment.py
- **作用**：评论与回复路由
- **功能**：
  - `/post/<int:post_id>/comment`：添加评论
  - `/comment/<int:comment_id>/reply`：回复评论（独立页面）
  - `/comment/<int:comment_id>/edit`：编辑评论
  - `/comment/<int:comment_id>/delete`：删除评论

#### routes/admin.py
- **作用**：管理员后台路由
- **功能**：
  - `/admin`：管理后台首页
  - `/admin/posts/pending`：待审核帖子列表
  - `/admin/post/<int:post_id>/approve`：通过审核
  - `/admin/post/<int:post_id>/reject`：拒绝审核
  - `/admin/reports`：举报列表
  - `/admin/report/<int:report_id>/handle`：处理举报
  - `/admin/users`：用户管理

#### routes/notice.py
- **作用**：公告路由
- **功能**：
  - `/notices`：公告列表页
  - `/api/notices`：公告 API（返回 JSON 数据）

#### routes/notification.py
- **作用**：通知路由
- **功能**：
  - `/notifications`：我的通知页面
  - `/api/notifications/unread-count`：未读通知数量
  - `/api/notifications/read/<int:notification_id>`：标记单条通知为已读
  - `/api/notifications/mark-read`：标记所有通知为已读
  - `/api/notifications/delete/<int:notification_id>`：删除通知

#### routes/report.py
- **作用**：举报路由
- **功能**：
  - `/post/<int:post_id>/report`：举报帖子

### templates/ 目录

- **base.html**：基础模板，包含导航栏、页脚、通用 JavaScript
- **index.html**：首页模板
- **search.html**：搜索页面模板
- **login.html**：登录页面模板
- **register.html**：注册页面模板
- **publish.html**：发布信息页面模板
- **post_detail.html**：帖子详情页面模板
- **edit_post.html**：编辑帖子页面模板
- **my_posts.html**：我的发布页面模板
- **notices.html**：公告列表页面模板
- **notifications.html**：通知页面模板
- **reply_comment.html**：回复评论页面模板
- **edit_comment.html**：编辑评论页面模板
- **admin_dashboard.html**：管理员后台首页模板
- **admin_reports.html**：举报管理页面模板

### static/ 目录

- **css/style.css**：自定义样式文件
- **js/main.js**：通用 JavaScript 脚本

## 设计原则

1. **单一职责原则**：每个文件只负责一个功能模块
2. **模块化**：路由按功能拆分为独立的蓝图
3. **分层架构**：
   - 表现层（routes/）：处理 HTTP 请求和响应
   - 业务层（services.py）：封装核心业务逻辑
   - 数据层（models.py）：数据库模型定义
   - 工具层（utils.py）：通用工具函数
4. **可维护性**：清晰的目录结构和文件命名，便于后续维护和扩展
