## 📊 我们使用的数据库：SQLite
### 一、SQLite 简介
SQLite 是一款 轻量级、嵌入式关系型数据库 ，由 D. Richard Hipp 于 2000 年发布。

### 二、核心特点
特点 说明 零配置 无需安装服务器，无需配置端口 文件式存储 整个数据库就是一个文件（ lost_found.db ） 跨平台 支持 Windows、Linux、macOS 等所有主流系统 轻量级 库文件仅几百KB，内存占用极低 事务支持 完全支持 ACID 事务 开源免费 公共领域软件，无版权限制

### 三、为什么选择 SQLite
对于校园失物招领系统，SQLite 是最佳选择 ，原因如下：

1. 开发便捷 ：不需要安装数据库服务器，直接使用
2. 部署简单 ：只需拷贝一个 .db 文件即可
3. 数据量合适 ：校园级数据量（几千到几万条）完全够用
4. 学习成本低 ：SQL 语法标准，容易上手
5. 适合课程项目 ：无需复杂配置，快速开发
### 四、在项目中的使用方式
数据库文件 ： lost_found.db

配置代码 （ app.py ）：

```
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///
lost_found.db'
```
使用的 ORM ：SQLAlchemy（Flask-SQLAlchemy）

### 五、数据库操作示例
创建表 ：

```
class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(80), unique=True)
    password_hash = db.Column(db.String(120))
```
查询数据 ：

```
posts = Post.query.filter_by(status='approved').all()
```
插入数据 ：

```
new_user = User(username='test', role='user')
new_user.set_password('123456')
db.session.add(new_user)
db.session.commit()
```
### 六、与其他数据库对比
数据库 类型 特点 适用场景 SQLite 嵌入式 轻量级、零配置 小型应用、开发测试 MySQL 客户端-服务器 功能强大、性能好 中型到大型应用 PostgreSQL 客户端-服务器 特性丰富、扩展性强 大型企业应用

### 七、优点与局限性
优点 ：

- ✅ 无需安装和配置
- ✅ 文件式存储，易于备份和迁移
- ✅ 性能足够支撑校园级数据量
- ✅ 完全支持 SQL 标准
局限性 ：

- ❌ 并发写入性能不如 MySQL
- ❌ 不适合超大规模数据（百万级以上）
- ❌ 没有用户权限管理（单文件访问）
### 八、总结
SQLite 是本项目的理想选择，它：

- 足够简单 ：适合快速开发课程项目
- 足够强大 ：支持所有必要的数据库操作
- 足够便携 ：一个文件即可部署
- 足够安全 ：本地文件访问，网络攻击面小
如果你以后需要扩展到更大规模，可以轻松迁移到 MySQL 或 PostgreSQL。
